# Permission Explainer / 权限解释器

AI-powered risk analysis plugin for Claude Code. Intercepts tool calls before execution, explains what's about to happen in natural language, and lets **you** make the final decision.

AI 驱动的 Claude Code 风险分析插件。在工具执行前拦截，用自然语言解释即将发生的操作，把最终决策权交给你。

## Why / 为什么需要

Claude Code can execute powerful system commands — deleting files, modifying configs, running scripts. The default permission dialog tells you *what* command will run, but not *what could go wrong*. This plugin adds a layer of risk awareness between the AI's suggestion and your approval.

Claude Code 可以执行强大的系统命令——删除文件、修改配置、运行脚本。默认权限弹窗告诉你*要执行什么*，但不会告诉你*可能出什么问题*。这个插件在 AI 建议和你批准之间，加了一层风险感知。

## Features / 功能

- **Smart Risk Analysis / 智能风险分析** — Local rule engine scores path + filename + operation scope. DeepSeek API independently evaluates, results are merged with weighted negotiation.
- **Natural Language Explanations / 自然语言解释** — "What", "Why", and "Worst case" generated by AI, not hardcoded templates.
- **Three-Option Interaction / 三选项交互** — Deny, Allow Once, or Always Allow (session-scoped trust).
- **Path-Aware Scoring / 路径感知** — System directories, credential files, important documents, temp files — all scored differently.
- **Cross-Platform / 跨平台** — Unix (`rm -rf /`, `sudo`, `chmod 777`) and Windows (`C:\`, `del /s`, `diskpart`) coverage.

## Risk Levels / 风险等级

| Level | Examples | Behavior |
|-------|----------|----------|
| 🟢 Low / 低 | `ls`, `echo`, `git status` | Auto-allow |
| 🟡 Medium / 中 | Edit/create files, delete temp files | Block + three options |
| 🟠 High / 高 | `rm`, `sudo`, delete .docx/.sql/backups | Block + three options |
| 🔴 Critical / 严重 | `rm -rf /`, disk ops, delete .env/keys | Block + three options |

## Install / 安装

1. Copy to `~/.claude/plugins/permission-explainer/`

2. (Optional) Configure DeepSeek API in `~/.claude/settings.json`:

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "your-api-key",
    "ANTHROPIC_MODEL": "deepseek-v4-pro"
  }
}
```
*Without API config, the plugin uses local rules only — still functional, just less nuanced.*

3. Add hook in `~/.claude/settings.local.json`:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash|Write|Edit|PowerShell|WebFetch",
        "hooks": [
          {
            "type": "command",
            "command": "python ${CLAUDE_PLUGIN_ROOT}/hooks/explain.py",
            "timeout": 20
          }
        ]
      }
    ]
  }
}
```

4. Restart Claude Code.

## How It Works / 工作原理

```
User action → PreToolUse Hook intercepts
            → Local rule engine scores (path + filename + scope)
            → DeepSeek API evaluates independently (optional)
            → Weighted negotiation merges results
            → Low risk: auto-allow | Medium+: block with three options
            → User chooses → Execute or deny
```

- **Safety floor / 安全底线**: Local rules and AI take the stricter assessment. False positive > false negative.
- **Session trust / 会话信任**: "Always Allow" resets when Claude Code restarts.
- **Self-protection / 自保机制**: Plugin's own files are never blocked, avoiding circular interception.
